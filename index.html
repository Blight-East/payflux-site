Before You Start

PayFlux runs as a secure container on your infrastructure.

You'll need Docker installed:

  • Mac / Windows: Docker Desktop
  • Linux: Docker Engine

If Docker runs, PayFlux runs.

One-Command Setup

Once Docker is installed, onboarding is a single command.

You'll connect your processor, choose your tier, and generate a ready-to-run setup — all from the dashboard.

$ payflux setup --tier 2

What happens next:

  • PayFlux starts locally in a secure container
  • A health check confirms everything is running
  • Warnings (Tier 2 + Pilot) appear in the dashboard automatically
  • You can stop everything instantly with one command

No agents. No system changes. Nothing runs outside Docker.

What You Get Immediately
  • HTTP ingestion endpoint at /ingest
  • Health check endpoint at /health
  • Warnings surface in the dashboard when detected
  • Optional alerting sidecar forwards warnings to Slack/webhooks (Tier 2)

That's it. No wizard fatigue. No YAML spelunking.

PayFlux surfaces signals and operational context; it does not block payments.
